1. O QUE É O NEXUSAGENT AI
================================================================================

Agente autônomo de programação com 15 provedores de IA gratuitos configurados.
Permite: gerar código, criar websites/apps/APKs, analisar arquivos, refatorar,
corrigir bugs, documentar, testar, deploy, e 13 outras tarefas.

7. ROTAS DA API
================================================================================

# Auth
POST /api/auth/register    → Criar conta
POST /api/auth/login       → Login (retorna JWT)
GET  /api/auth/me          → Dados do usuário logado
GET  /api/auth/api-keys    → Listar keys do usuário
PUT  /api/auth/api-keys    → Salvar keys

# Tasks
POST /api/tasks/execute-raw → Executar tarefa (com upload de arquivo)
GET  /api/tasks/history     → Histórico de tarefas
GET  /api/tasks/{id}        → Detalhe de tarefa

# Projects
GET  /api/projects          → Listar projetos
POST /api/projects          → Criar projeto
GET  /api/projects/templates → Templates disponíveis

# System
GET  /api/health            → Status do servidor
GET  /api/stats             → Estatísticas

10. COMO ADICIONAR NOVO PROVEDOR
================================================================================

1. Abrir /root/agent/backend/config.py
2. Adicionar entrada em FREE_AI_PROVIDERS:
   "novo_provedor": {
       "name": "Nome",
       "url": "https://api.exemplo.com/chat/completions",
       "models": ["model-1", "model-2"],
       "free_tier": True,
       "speed": "fast",
       "strengths": ["code", "general"],
       "api_key_env": "NOVO_API_KEY",
   }
3. Adicionar em TASK_ROUTING e COST_OPTIMIZATION_ORDER
4. Reiniciar servidor

16. SOLUÇÃO DE PROBLEMAS
================================================================================

# Servidor não inicia:
- Verificar se porta 8000 está livre: lsof -i :8000
- Verificar logs: cat /tmp/nexus*.log

# IA não responde:
- Verificar se tem key configurada (Configurações no site)
- Testar key: curl -X POST https://openrouter.ai/api/v1/chat/completions \
  -H "Authorization: Bearer SUA_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"cohere/north-mini-code:free","messages":[{"role":"user","content":"hi"}]}'

# Tunnel não funciona:
- Reiniciar: pkill cloudflared && cloudflared tunnel --url http://localhost:8000 --no-autoupdate
- URL muda a cada reinício (limitação do plano gratuito)
